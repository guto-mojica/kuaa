{# ── Shared field-level error fragment (U1) ────────────────────────────────── Accessible inline validation message for a single input surface, designed to be returned as an HTMX out-of-band swap so the message lands in a stable error slot next to the offending field WITHOUT a full-page reload and WITHOUT clobbering the field's normal swap target (e.g. #search-results). Caller contract: * ``slot_id`` — id of the always-present empty error slot in the page (e.g. "search-query-error"). This element's id is what the field's ``aria-describedby`` points at, so screen readers announce the message and associate it with the input. * ``message_key`` — a STABLE machine key (see the map below). The human string is translated HERE so the literals live in a template (the Babel catalog is template-only; see web/babel.cfg) and PT/EN both resolve. Empty / unset key → an empty slot (the clear-error path). Optional: * ``oob`` — when true (the HTMX path), the element carries its own ``id=""`` and ``hx-swap-oob="outerHTML"`` so HTMX matches the live slot by id and replaces it regardless of the request's primary target. When false (server-side first render / no-JS), the bare slot is emitted inline. a11y: * ``role="alert"`` + ``aria-live="assertive"`` → the message is announced the moment it appears. These are emitted ONLY when a message is present (U5): an always-on, empty assertive live region is a known screen-reader nuisance — some ATs announce the region or fire a spurious "alert" on page load / on every unrelated OOB swap that re-renders the empty slot. Gating the live-region roles on ``message`` keeps the slot's ``id`` stable for ``aria-describedby`` (the element is always in the DOM) while only turning it into an alert at the moment it actually carries text. When the route clears the error it swaps back the role-less empty span, which silently removes the live region again. * The ``data-field-error`` marker lets the htmx:beforeSwap shim in mojica.js permit this fragment to swap even on a 4xx response (HTMX suppresses body swaps on error codes by default). #} {# Key → translated string.
