{# ── Toolrow: Appearance / Fields / Filters / Grouped by / Sorted + .find ─ The toolrow is the HTMX include scope: every refresh that hits ``/api/scenes`` (keyword change on ``.find``, tag toggle in the Filtros popover) sends ``hx-include="#scenes-toolrow"`` so both the keyword AND the active tag selection ride along. Keeping the inputs co-located is what makes ``q + film + tags[]`` survive partial swaps without a global form. Appearance + Fields are pure client-side prefs driven by Alpine stores in ``mojica.js`` (``$store.cenasAppearance`` / ``$store.cenasFields``). They flip class names on the wrapping ``.c-cp`` section above; CSS in ``cenas.css`` translates those classes into density + per-field visibility overrides. State is persisted to ``localStorage`` so a refresh keeps the user's chosen card density and hidden fields. #}
