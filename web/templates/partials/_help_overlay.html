{#- Keyboard help overlay (Phase 7 / Task 28). Rendered once into base.html at page load (closed); ``mojica.js`` registers an Alpine store (``Alpine.store('help', { open: false })``) on alpine:init and ``window.Help.{open,close,toggle}`` flip that store. The markup below reacts via ``x-show="$store.help.open"``, so opening the overlay is one assignment instead of a DOM walk. The DOM stays server-rendered (no HTMX) so the first frame after the user mashes ``?`` paints the legend without any network round trip. Class names follow web/static/css/polish.css verbatim: * ``.kh-back`` — backdrop (fixed, full-viewport, blurred) * ``.kh-panel`` — centred panel (720px max) * ``.kh-head``, ``.kh-head h2``, ``.kh-head h2 .pip`` (the "?" badge), ``.kh-head .close`` (the dismiss button) * ``.kh-body`` — 2-column grid container * ``.kh-group`` — one column of shortcuts under an h3 heading * ``.kh-row`` — single shortcut row (description + keys) * ``.kh-row .desc``, ``.kh-row .keys``, ``.kh-row .keys .k``, ``.kh-row .keys .sep`` Alpine bindings on the root element: * ``x-show="$store.help.open"`` + ``x-cloak`` — reactive visibility; x-cloak keeps the overlay hidden during the brief window before Alpine processes the DOM on first paint. * ``@click.self="$store.help.open = false"`` — backdrop click dismisses. ``.self`` ensures the handler only fires when the click target IS .kh-back (i.e. the user clicked outside the panel); clicks inside .kh-panel hit one of its descendants and bypass this handler entirely. * ``@keydown.escape.window`` — Esc dismisses, but only when help is the active surface. ``.window`` is required because focus is rarely on the overlay element itself when Esc is pressed. ``$event.preventDefault()`` runs only when help is open so the keystroke can still propagate to other handlers (palette's own Esc, future surfaces) when help is closed. Mutual exclusion with the command palette is enforced on the JS side (see ``window.Help`` and the ⌘K block in mojica.js): opening one closes the other, so a tap of ``?`` while the palette is open swaps the two surfaces cleanly. The legend lists only shortcuts wired in the current branch. Keep this strict: if a key is visible here, it should have a backing handler. -#} {% from "partials/_icons.html" import icon %}
