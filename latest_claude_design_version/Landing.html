Mojica Cinemateca · Multimodal retrieval · v1.0

Searching the moving image, across decades, locally.

Cinemateca Mojica is an offline-first multimodal retrieval engine for film archives. Six early Brazilian films, 1.588 scene-level keyframes, indexed against CLIP, BM25, and a cross-encoder reranker — runs entirely on a single GPU, no cloud APIs.

02 What this is

A working tool, not a product.

Mojica is the internal cataloguing system for an independent Brazilian cinematheque — a small institution that holds 35mm prints, mid-century talkies, museum-grade documentaries. The job is mundane and important: find a scene by what's in it, across the entire acervo, in seconds.

"The interface is a tool for professionals. It should disappear in service of the material."

The previous version of this tool was a 412-row spreadsheet and a folder of JPEGs. This version indexes every scene against CLIP-L/14 visual embeddings, generates a description per keyframe with Moondream 2, transcribes dialogue with Whisper-large-v3, and lets a curator query across modalities — type a phrase, paste an image, sing a melody, find the scene.

The signature feature for v1.0 is visual rhymes: a kNN over the visual-embedding space with a cross-film constraint, designed to surface formal echoes between, say, a Mazzaropi long-shot from 1959 and a Mário Peixoto silhouette from 1931. It's how a curator builds a programme.

Everything runs locally. No cloud APIs, no telemetry, no external dependencies in the hot path. The whole catalogue fits in a 6 GB SQLite + FAISS index on a workstation, and the demo you can try above is the same backend the curators use.

03 Signature feature · Rimas visuais

Cross-film visual rhymes.

A kNN over CLIP-L/14 embeddings with a cross-film constraint and MMR-diversity reranking. Given an anchor scene, surface its formal echoes — across decades, across directors, across stocks. Visual cinema, not metadata-tagged cinema.

04 Pipeline

How it works.

A five-stage offline pipeline turns a raw .mp4 into a queryable index in roughly twice real-time on a single T4.

01

Frames

Extract one frame per second, plus shot-boundary keyframes.
FFmpeg · PySceneDetect
02

Cenas

Detect cuts; group frames into scene units with timecodes.
PySceneDetect · content-aware
03

Visual

Detect objects, faces, count people; generate auto-tags.
YOLOv8 · MTCNN
04

Embeddings

CLIP visual + text embeddings; Whisper transcription embeddings.
CLIP-L/14 · Whisper-large-v3 · FAISS IVF1024
05

Descrições

VLM caption per keyframe; index into BM25 lexical store.
Moondream 2 · rank_bm25

At query time, three retrievers run in parallel — semantic (cosine over CLIP-L/14), lexical (BM25 over Moondream descriptions + Whisper transcripts), and an mxbai-rerank-large cross-encoder over the topk union. Results are fused with a tunable weight (defaults to 0.7 / 0.3 sem/bm25 then rerank) and diversified with MMR. The whole hot path is sub-300ms for a 6-film catalogue.
