Aprofunde-se · Proveniência da execução

Todo registro de catálogo rastreia de volta a uma execução.

Para revisores técnicos, a reprodutibilidade importa tanto quanto a precisão. Toda execução do pipeline escreve um recibo legível por máquina — run_manifest.json — que responde: qual entrada, qual configuração, e quais versões de modelo produziram este catálogo específico?

O manifesto de execução

O que é registrado, em toda execução.

Um auxiliar de manifesto compartilhado escreve o mesmo registro após sucesso, erro, uma etapa seguinte bloqueada, ou cancelamento pelo usuário — não apenas em uma execução limpa. A escrita do manifesto é best-effort: uma falha de escrita é registrada em log, mas nunca substitui o resultado real do pipeline.

  • caminho de entrada, existência, tamanho e data de modificação
  • um hash SHA-256 de configuração mais um snapshot de configuração compatível com JSON
  • o id, o rótulo e o caminho do pacote de domínio selecionado
  • backends de modelo configurados e dicas de versão de modelo
  • estados, durações, erros e caminhos de saída por etapa
  • caminhos de artefatos de saída esperados e se cada um existe
  • timestamps de início/fim da execução e status terminal

Exemplo (formato ilustrativo)

{
  "input": {
    "path": "/data/raw/jeca_tatu_1959.mp4",
    "exists": true,
    "size_bytes": 812345678
  },
  "config_hash": "sha256:a9b3c4d1f...",
  "domain": { "id": "archive", "label": "Film archive" },
  "model_backends": {
    "image_embedder": "siglip_multilingual",
    "scene_describer": "moondream_transformers",
    "object_detector": "yolov8"
  },
  "steps": {
    "frame_extraction": { "status": "done", "duration_s": 42.1 },
    "llm_description": { "status": "done", "duration_s": 5123.4 }
  },
  "terminal_status": "done"
}

Impacto operacional

Por que isso importa além da auditoria.

Rastreável, não apenas registrado em log

Como todo campo é estruturado e toda execução escreve um, uma mudança de configuração ou troca de modelo que altera a qualidade da recuperação pode ser rastreada de volta até a execução exata que a causou — não reconstruída de memória ou de linhas de log espalhadas.

O detalhamento completo campo a campo está em docs/OPERATIONS.md.
