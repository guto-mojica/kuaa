Go deeper · Evaluation

Measuring retrieval quality, not asserting it.

KUAA moves beyond "does it work?" with a repeatable measurement harness: a reviewed query dataset for the public archive demo, deterministic retrieval metrics over an existing index, and correction statistics that compare generated tags with manual annotations.

Fixed test bed

Same queries, same index, every run.

Controlled evaluation

Query sets are versioned YAML files with required id, text, and relevant_scene_ids fields per query, plus optional intent notes, graded relevance, and negative examples. The public archive demo set lives at data/eval/archive_demo_queries.yaml.

The harness scores an existing retrieval index — it does not upload video, frames, metadata, or embeddings anywhere. It runs the same way against the public demo bundle or a private local collection.

Implemented metrics

Standard information-retrieval metrics.

MetricDefinitionWhat it measures
Recall@5Mean per-query fraction of expected scenes found in the top 5Breadth — did we find most of what was relevant?
Recall@10Mean per-query fraction of expected scenes found in the top 10Same, at a wider cutoff
MRRMean reciprocal rank of the first relevant resultHow quickly a relevant item is found
nDCG@10Normalized discounted cumulative gain at 10Whether the best results rank highest, not just whether they appear

Retrieval can be scored across text, image, and cross-film-rhyme modalities. For ungraded queries, all relevant scenes receive the same grade; queries without any relevant scenes are rejected by validation rather than silently skipped.

Human-curator signal

What did manual review actually improve?

Generated tags and manual annotations are stored separately, then compared after normalization. The correction-stats report is a proxy signal — it measures agreement and additions, not whether every unaccepted generated tag was actually wrong — but it gives a concrete, countable answer to "what did human review change?" instead of a vague claim that review "helps."

Correction statistics

The comparison reports scenes with manual annotations, generated tags accepted by manual annotation, human-added tags absent from generated metadata, generated tags the curator dropped, and an overall correction rate.

Numeric results are only published once they're run against the final, verified demo artifact bundle — see docs/EVALUATION.md for the full harness and CLI commands.
